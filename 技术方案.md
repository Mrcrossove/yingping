# 客户下单小程序与后台管理收口技术方案

## 目标

把系统从“多角色小程序 + 后台管理”收口为：

- 小程序：只面向商家/客户，用于微信登录、添加地址和手机号、下单、支付、查看订单。
- 后台：面向老板、管理员、业务员、制作员、配送员、推广员，用于订单处理、员工管理、推广管理、分成提现等。

这样可以减少客户下单阻塞，避免商户入驻审核和员工小程序工作台干扰真实业务。

## 新业务流程

### 客户/商家

1. 打开小程序。
2. 微信登录。
3. 自动创建 `merchant` 用户，默认启用。
4. 添加手机号和收货地址。
5. 选择商品加入购物车。
6. 下单。
7. 支付。
8. 查看自己的订单。

### 推广员

1. 推广员登录后台。
2. 后台生成/查看自己的推广码和小程序码。
3. 推广员把小程序码给客户扫码。
4. 客户扫码进入小程序，登录后自动绑定推广员。
5. 客户后续下单完成后，推广员获得分成。
6. 推广员在后台查看推广商家数量、绑定商家、分成记录。

### 老板/员工

全部在后台操作，不在小程序操作：

- 老板/管理员：商品、订单、员工、权限、提现、支付、推广管理。
- 业务员：后台接单、派单、代客录单。
- 制作员：后台查看和处理制作任务。
- 配送员：后台查看和处理配送任务。
- 推广员：后台查看推广码、推广商家和分成。

## 当前系统偏移与处理

### 1. 商户入驻流程

当前存在：

- 小程序商家入驻页面。
- 账号密码注册。
- `status = 2` 待审核。
- 后台商户审核。

新方案处理：

- 小程序隐藏商家入驻入口。
- 客户只通过微信登录创建 `merchant` 用户。
- 微信登录创建的用户默认 `status = 1`。
- 保留后台商户审核页面和注册接口作为备用管理功能，但不作为客户主入口。

### 2. 小程序多角色入口

当前存在：

- 员工工作台。
- 推广中心。
- 收益明细。
- 提现申请。
- 账号密码登录。
- 绑定推广码菜单。

新方案处理：

- 小程序“我的”只展示客户功能：
  - 微信登录。
  - 我的订单。
  - 收货地址。
  - 消息通知。
  - 退出登录。
- 不再展示：
  - 账号密码登录。
  - 商家入驻。
  - 绑定推广码。
  - 推广中心。
  - 员工工作台。
  - 收益明细。
  - 提现申请。
- 首页隐藏业务员代客录单入口。

### 3. 推广绑定

当前已有基础：

- `PromotionCode`。
- `MerchantBinding`。
- `AuthService.wxLogin(code, promoterCode?)`。
- 小程序 `capturePromoterCode()`。
- 完成订单后按推广员计算分成。

新方案处理：

- 保留扫码 scene / promoterCode 捕获。
- 客户微信登录时将推广码传给后端。
- 后端只在未绑定时绑定，已绑定不覆盖。
- 后台推广页增强：
  - 推广员能查看自己的推广码。
  - 推广员能获取小程序码。
  - 推广员能查看已绑定商家数量。
  - 推广员能查看分成记录。

## 实施步骤

### 阶段 1：小程序收口为客户下单端

- 修改 `pages/mine/mine.vue`：
  - 移除账号密码登录入口。
  - 移除多角色菜单和收益显示。
  - 只保留客户菜单。
- 修改 `pages/index/index.vue`：
  - 隐藏业务员代客录单入口。
- 保留员工/推广相关页面文件，但不暴露入口。

验收标准：

- 小程序普通用户只看到下单相关能力。
- 没有员工、推广、收益、提现、商家入驻入口。

### 阶段 2：后台推广员能力增强

- 修改后台 `Promotion.vue`：
  - 推广员可查看自己的推广码。
  - 推广员可生成推广码。
  - 推广员可查看小程序码。
  - 推广员可查看绑定商家数量。
  - 推广员可查看分成记录。
- 后端补充推广统计返回：
  - 绑定商家数。
  - 分成总额。

验收标准：

- 推广员无需进入小程序，也能在后台完成推广码和数据查看。

### 阶段 3：后端登录语义确认

- 微信登录新用户默认 `merchant` 且 `status = 1`。
- 微信登录时自动绑定推广码。
- 已绑定推广员不允许覆盖。
- 账号密码注册/商户审核保留为备用，不在小程序展示。

验收标准：

- 客户扫码后微信登录即可下单。
- 推广绑定只发生一次。

## 不在本轮处理

- 微信支付正式闭环：仍依赖商户号、API v3 key、商户 API 证书、小程序合法域名。
- 删除数据库表：保留 `merchant_profiles`、商户审核相关表和页面，避免破坏已有数据。
- WebSocket 实时消息：后续增强。
