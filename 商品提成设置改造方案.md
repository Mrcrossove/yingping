# 商品提成设置改造方案

## 背景

当前系统存在两个问题：

- 商品新增/编辑弹窗里有“制作提成(元)”和“配送提成(元)”字段，但真实结算并不使用这两个字段，容易误导后台用户。
- 当前提成规则按“商品分类 + 角色 + 百分比”设置，不符合实际经营需要。老板希望对每一款商品分别设置不同角色提成，并且支持百分比和固定金额两种方式。

## 目标

把提成规则改成：

- 按具体商品设置，而不是按分类设置。
- 每个商品支持设置四类角色提成：
  - 业务员 `salesperson`
  - 制作员 `maker`
  - 配送员 `delivery`
  - 推广员 `promoter`
- 每个角色支持两种提成方式：
  - 百分比：按商品成交金额百分比计算。
  - 固定金额：按商品数量乘以固定金额计算。

## 计算规则

订单完成后，逐商品、逐角色计算提成：

```text
百分比提成 = 商品成交单价 × 数量 × 提成值 / 100
固定金额提成 = 提成值 × 数量
```

员工归属：

- 业务员：订单 `salespersonId`
- 制作员：订单 `makerId`
- 配送员：订单 `deliveryId`
- 推广员：商户绑定的 `promoterId`

未配置规则或提成值为 0 时，不生成收益。

## 数据库改造

保留原表名 `commission_rules`，降低接口和部署复杂度。

旧结构：

```text
category_id
role
percentage
```

新结构：

```text
product_id
role
type      percentage / fixed
value
```

唯一约束：

```text
product_id + role
```

迁移策略：

- 新增 `product_id`、`type`、`value` 字段。
- 将旧的分类百分比规则按当前商品展开成商品规则。
- 删除旧的 `category_id`、`percentage` 字段和分类唯一索引。
- 老分类规则无法覆盖未来新商品，新商品需要在后台单独设置提成。

## 后端改造

### 提成设置接口

- `GET /api/commissions`
  - 返回商品列表及每个商品的角色提成规则。
- `POST /api/commissions/product/:productId`
  - 批量保存某个商品四个角色的提成规则。
- `DELETE /api/commissions/:id`
  - 删除单条角色规则，保留兼容。

### 订单结算

修改 `calculateCommissions()`：

- 从按 `categoryId` 查规则改成按 `productId` 查规则。
- 根据 `type` 决定百分比或固定金额计算。
- 同一订单仍然只结算一次，避免重复生成收益。

## 后台改造

### 商品管理

- 移除“制作提成(元)”字段。
- 移除“配送提成(元)”字段。
- 保留库存、价格、单位、描述等商品基础信息。

### 提成设置

把页面改为“商品提成设置”：

- 支持按分类筛选。
- 支持按商品名称搜索。
- 表格按商品展示：
  - 商品名称
  - 分类
  - 单价
  - 业务员提成
  - 制作员提成
  - 配送员提成
  - 推广员提成
  - 操作
- 点击“设置”后，一个弹窗内编辑该商品四个角色的提成。
- 每个角色可选择：
  - 不设置
  - 百分比
  - 固定金额

## 验收标准

- 商品弹窗不再出现无效提成字段。
- 每款商品能单独设置四个角色提成。
- 支持百分比和固定金额。
- 订单完成后按商品规则正确生成收益。
- 未设置规则的角色不生成收益。
- 后台和后端构建通过。
