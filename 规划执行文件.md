# 员工收益后台闭环规划执行文件

## 目标

让后台管理系统支持每个员工角色查看自己的业务收益和提现进度，同时让老板/管理员可以查看全员收益总账，形成：

- 员工处理自己的订单任务。
- 系统在订单完成后自动生成提成收益。
- 员工在后台查看收益、申请提现、查看提现记录。
- 老板/管理员在后台查看全员收益明细、导出、审核提现。
- 提成规则口径清晰，不让商品字段和分类提成规则产生误导。

适用员工角色：

- 业务员 `salesperson`
- 制作员 `maker`
- 配送员 `delivery`
- 推广员 `promoter`

老板 `boss` 和管理员 `admin` 主要看全局数据与审核，不作为普通员工收益看板的默认对象。

## 阶段 1：员工我的收益页面

### 范围

- 新增后台页面 `admin/src/views/MyEarnings.vue`。
- 接入已有接口：
  - `GET /api/earnings/my`
  - `GET /api/withdrawals/my`
  - `POST /api/withdrawals/apply`
- 页面展示：
  - 累计待结算收益
  - 已占用/已提现金额
  - 可提现金额
  - 收益明细
  - 提现申请表单
  - 我的提现记录

### 权限与入口

- 允许 `salesperson`、`maker`、`delivery`、`promoter` 访问。
- 后台菜单新增“我的收益”。
- 员工登录后仍可进入订单/推广业务页面，但可以从菜单进入收益看板。

### Review 点

- 员工不能访问老板的提现审核页。
- 商户不显示该入口。
- 提现金额不能大于可提现金额。
- 页面空数据时不报错。

## 阶段 2：老板/管理员员工收益总账

### 范围

- 新增后台页面 `admin/src/views/Earnings.vue`。
- 接入已有接口：
  - `GET /api/earnings`
  - `GET /api/users`
  - `GET /api/export/earnings`
- 页面展示：
  - 员工收益明细列表
  - 员工、角色筛选
  - 导出入口

### 权限与入口

- 允许 `boss` 访问。
- 允许拥有 `finance:view` 权限的 `admin` 访问。
- 菜单新增“员工收益”。

### Review 点

- 普通员工不能访问总账。
- 管理员没有 `finance:view` 时不显示菜单。
- 导出入口不影响已有提现审核导出。

## 阶段 3：提成规则与商品字段口径修正

### 范围

- 当前真实结算逻辑使用“分类 + 角色百分比”提成规则。
- 商品管理中的“制作提成(元)”和“配送提成(元)”当前不参与结算。
- 本阶段先在商品页面标注为“备用字段”，避免误导。
- 后续如要按单品固定金额提成，需要单独改造后端结算规则。

### Review 点

- 页面文案不再让老板误以为商品字段已参与提成。
- 不改数据库结构，避免影响线上数据。

## 阶段 4：数据看板接口修复

### 范围

- 修复后台首页中原生 `fetch('/api/...')`。
- 改为统一使用 `dashboardApi`，走 `API_BASE_URL` 和认证拦截器。

### Review 点

- 跨域名部署时看板接口仍能正常请求。
- 不破坏已有首页数据卡片。

## 阶段 5：构建与最终 Review

### 验证命令

```bash
cd admin
npm run build
```

```bash
cd server
npm run build
```

### 最终检查

- 员工角色能看到“订单管理/我的收益”。
- 推广员能看到“推广管理/我的收益”。
- 老板能看到“员工收益/提现审核/提成设置”。
- 商户不进入后台员工收益功能。
- 后台构建通过。
