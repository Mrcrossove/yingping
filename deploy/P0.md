# P0 — 核心业务流程修复

> 基于 `饮品.docx` + `新建 DOCX 文档.docx` 需求缺口分析  
> 实施时间: 2026-05-31

## P0-1 派单同时分配制作员+配送员

**需求来源**: PRD §4 — 业务员点击派单，系统需支持选择制作员+配送员

**实现**:
- `server/src/modules/order/order.service.ts` — 新增 `dispatchBoth(orderId, makerId, deliveryId)` 方法
- `server/src/modules/order/order.controller.ts` — 新增 `POST /orders/:id/dispatch-both` 路由
- `admin/src/api/order.ts` — 新增 API 方法
- `admin/src/views/OrderDetail.vue` — 派单 UI 同时选制作员+配送员

## P0-2 制作员"开始制作"操作

**需求来源**: PRD §3.2.2 — Maker 有"开始制作"→"确认完成"两步

**实现**:
- `server/src/modules/order/order.service.ts` — 新增 `makerStartMaking(orderId, makerId)` 方法
- `server/src/modules/order/order.controller.ts` — 新增 `POST /orders/:id/maker-start` 路由
- `miniapp/src/pages/employee/employee.vue` — 制作员视图增加"开始制作"按钮
- `miniapp/src/pages/order-detail/order-detail.vue` — 同步增加按钮

## P0-3 配送员"去配送"操作

**需求来源**: PRD §3.2.3 — Delivery 有"去配送"→"确认送达"两步

**实现**:
- `server/src/modules/order/order.service.ts` — 新增 `deliveryStartDelivering(orderId, deliveryId)` 方法
- `server/src/modules/order/order.controller.ts` — 新增 `POST /orders/:id/delivery-start` 路由
- `miniapp/src/pages/employee/employee.vue` — 配送员视图增加"去配送"按钮
- `miniapp/src/pages/order-detail/order-detail.vue` — 同步增加按钮

## P0-4 WebSocket 实时推送集成

**需求来源**: PRD §4 — 派单后指令实时推送到员工小程序端

**实现**:
- `server/src/modules/order/order.service.ts` — 注入 WebsocketGateway，各状态变更方法中调用 notify
- `miniapp/src/pages/employee/employee.vue` — 接入 WebSocket 监听新任务通知

## P0-5 订单 completed 终态 + 自动结算

**需求来源**: PRD §4 — 订单闭环，系统根据规则计算收益

**实现**:
- `server/prisma/schema.prisma` — OrderStatus 枚举增加 `completed`
- `server/src/modules/order/order.service.ts` — deliveryComplete 自动流转到 completed
