# 未完成功能技术实施方案

## 目标

把当前项目中“接口存在但没有业务闭环”“页面存在但没有真实运营能力”“权限数据存在但没有实际生效”的功能补齐，按阶段交付，每个阶段完成后都进行构建和代码 review，发现问题立即修复。

本轮覆盖：

1. 通知系统事件来源。
2. 管理员权限分配真正生效。
3. 推广商家线索后台管理。
4. 订单列表再次下单。
5. 评价功能小程序入口和后台查看。
6. 地址页面地区选择体验。
7. 清理小程序 mock 兜底和 mock 角色 Store。
8. 导出功能补齐筛选和收益入口。
9. 后台支付记录退款入口。
10. 商品库存预警和库存运营入口。
11. WebSocket 作为后续增强，不阻塞本轮核心上线。

微信支付正式 JSAPI 支付和微信 v3 回调验签需要商户证书、API v3 key、合法域名等资料齐全后单独实施，不纳入本轮强依赖。

## 阶段 1：运营后台闭环

### 1.1 通知事件来源

后端改动：

- 在订单流程中写入通知：
  - 商户下单后通知业务员/管理员。
  - 业务员接单后通知商户。
  - 派单给制作员/配送员后通知对应员工。
  - 制作完成后通知业务员/配送员。
  - 配送完成后通知商户。
- 在提现流程中写入通知：
  - 用户提交提现后通知管理员。
  - 审核通过、拒绝、标记打款后通知申请用户。
- 在商户审核流程中写入通知：
  - 商户提交申请后通知管理员。
  - 审核通过/拒绝后通知商户。

验收标准：

- `pages/notifications/notifications.vue` 不再长期为空。
- 对订单、提现、商户审核进行操作后，对应用户能看到通知。

### 1.2 推广商家线索后台管理

后端改动：

- 新增接口：
  - `GET /api/promotion/merchant-leads`
  - `PUT /api/promotion/merchant-leads/:id/status`
- 老板/管理员可查看全部线索，推广员只查看自己的线索。

后台改动：

- `Promotion.vue` 新增“上传商家线索”Tab。
- 支持查看商家名称、电话、地址、备注、推广员、状态。
- 支持管理员把状态改为：待跟进、已跟进、已转化、已拒绝。

验收标准：

- 小程序推广员上传的商家线索能在后台看到。
- 后台修改状态后，小程序推广员再次进入推广页能看到最新状态。

## 阶段 2：权限和小程序业务闭环

### 2.1 权限分配真正生效

后端改动：

- 将 `PermissionsGuard` 加入需要细粒度控制的后台接口。
- 给关键接口加 `@Permissions()`：
  - 订单管理：`order:manage`
  - 派单管理：`order:dispatch`
  - 员工管理：`user:manage`
  - 商品管理：`product:manage`
  - 提成设置：`commission:manage`
  - 提现审核：`withdrawal:manage`
  - 数据导出：`export:manage`
  - 推广管理：`promotion:manage`
  - 财务数据：`finance:view`

后台改动：

- 保留老板直接拥有所有权限。
- 管理员根据分配权限显示菜单和按钮。

验收标准：

- 没有对应权限的管理员访问接口会被拒绝。
- 没有对应权限的管理员不显示对应菜单。

### 2.2 订单列表再次下单

小程序改动：

- `orders.vue` 的再次下单读取订单商品并加入购物车。
- 跳转到购物车而不是仅回首页。

验收标准：

- 完成订单点击再次下单后，购物车出现原订单商品和数量。

### 2.3 评价功能入口

小程序改动：

- 订单详情页对已完成订单显示评价入口。
- 支持评分、文字评价。
- 已评价订单不重复评价。

后台改动：

- 新增评价管理页或在订单详情页展示评价。

验收标准：

- 商户能对已完成订单评价。
- 后台能查看评价内容。

## 阶段 3：体验和运营补齐

### 3.1 地址地区选择器

小程序改动：

- 地址编辑页使用地区选择器替代手动省市区输入。
- 保留详细地址输入。

验收标准：

- 用户选择省市区后自动填充字段。

### 3.2 清理 mock 兜底

小程序改动：

- 首页商品/分类接口失败时显示空状态或错误提示，不再展示 mock 商品。
- 移除 `mockRole` 相关正式逻辑。
- 状态字典从 `mock` 文件迁移到独立常量文件。

验收标准：

- 线上接口失败不会展示假商品、假分类、假角色。

### 3.3 导出功能补齐

后端改动：

- 订单导出支持 `status`、`keyword`、`startDate`、`endDate`。
- 收益导出支持角色/用户筛选。

后台改动：

- 收益/财务相关页面增加导出入口。

验收标准：

- 后台页面筛选条件和导出的 Excel 数据一致。

### 3.4 支付记录退款入口

后台改动：

- 支付记录页对已支付订单显示“退款”按钮。
- 调用后端 `POST /api/payments/refund/:orderId`。

后端说明：

- 当前后端退款只是状态变更，不是微信原路退款。
- 正式微信退款需要等微信支付资料齐全后单独做。

验收标准：

- 后台能把已支付记录标记为已退款。

### 3.5 商品库存运营

后台改动：

- 商品列表展示库存和库存预警。
- 低库存商品突出提示。
- 支持编辑库存、最低库存。

验收标准：

- 商品库存低于最低库存时后台可识别。

## 阶段 4：WebSocket 增强

当前 WebSocket 不阻塞业务上线。后续增强：

- 小程序登录后连接 WebSocket。
- 后台工作台连接 WebSocket。
- 新订单、派单、状态变化实时提醒。

验收标准：

- 不刷新页面也能收到新订单/新任务提醒。

## Review 规则

每个阶段完成后执行：

```bash
cd server
npm run prisma:generate
npm run build
```

```bash
cd admin
npm run build
```

```bash
cd miniapp
npm run build:mp-weixin
```

Review 检查项：

- 接口权限是否过宽。
- 页面是否仍显示 mock/假数据。
- 前后端字段是否一致。
- 新增接口是否有角色限制。
- 构建是否通过。
- 数据库迁移是否兼容已有数据。
