# 开发计划执行清单

## 优先级 P0：支付闭环

目标：把“支付暂未开通”改成可接入微信 JSAPI 支付的标准链路。

开发任务：

- 小程序订单详情接入 `paymentApi.create`、`paymentApi.jsapi`、`uni.requestPayment`。
- 后端支付回调改为微信支付 v3 通知格式，完成资源解密和幂等处理。
- 支付成功后更新支付状态，并把订单从 `pending` 推进到可接单状态。
- 退款接口改为微信支付退款接口调用，不能只改本地状态。
- 后台支付记录保留退款入口，但提示改为真实退款。

后续线上 `.env` 需要补齐：

```env
WX_APPID=""
WX_SECRET=""
WX_MCH_ID=""
WX_SERIAL_NO=""
WX_PRIVATE_KEY_PATH="/opt/beverage-order/server/certs/apiclient_key.pem"
WX_PLATFORM_CERT_PATH="/opt/beverage-order/server/certs/wechatpay_platform.pem"
WX_API_V3_KEY=""
WX_PAY_NOTIFY_URL="https://api.gzzzyp.com/api/payments/notify"
```

服务器文件：

```text
/opt/beverage-order/server/certs/apiclient_key.pem
/opt/beverage-order/server/certs/wechatpay_platform.pem
```

## 优先级 P1：库存闭环

目标：库存字段真正参与下单和取消。

开发任务：

- 下单时校验库存是否足够。
- 下单成功扣减库存。
- 取消未完成订单时回滚库存。
- 后台订单详情展示库存相关异常时给出明确错误。

## 优先级 P1：收益/提现结算闭环

目标：提现打款后收益状态可追踪，避免财务口径混乱。

开发任务：

- 新增提现和收益之间的明细关联表。
- 提现申请时锁定一批可提现收益。
- 审核拒绝时释放锁定。
- 标记打款时把关联收益标记为 `withdrawn`。

## 优先级 P2：订单状态口径统一

目标：统一“已送达”和“已完成/完结”的业务含义。

开发任务：

- 保留现有枚举兼容历史数据。
- 明确提成计算以 `completed` 为最终结算态。
- 配送员送达后进入 `delivered`，支付/人工确认后可进入 `completed`。
- 当前如无二次确认需求，可暂时把送达同时置为 `completed`。

## 优先级 P2：后台订单地址地图和导出增强

目标：后台运营能看到完整收货信息，导出能用于配送和对账。

开发任务：

- 后台订单详情展示收货人、电话、地址、地图选点名称、经纬度。
- 后台订单详情提供复制地址。
- 订单导出增加收货信息、联系电话、支付状态。

## 优先级 P3：清理残留页面和通知增强

目标：减少无效代码和补齐运营提醒。

开发任务：

- 清理或归档小程序不可路由的旧员工/入驻页面。
- 清理后台未挂载的商户审核页面，或明确保留为历史兼容。
- 补支付成功、退款、库存预警、评价通知。
